# 급락 조기 대응 트리거 (Rapid Decline Trigger)

## 배경 / 문제
기존 레짐 판단(`detect_market_regime`)은 **SMA200 + 카나리아(월 단위 모멘텀) + HMM** 기반이라
**느린 지표**다. 하루~며칠 만의 급락(crash)에는 BEAR/CRISIS 전환이 지연되어,
방어·인버스 대응이 늦을 수 있다.

## 목표
지수(KOSPI 프록시 = KODEX 200, US = SPY)의 **1일·3일 누적 급락률**로 빠르게 위험을 감지해
레짐을 조기에 **더 방어적으로만** 격상한다.

## 핵심 설계 원칙
1. **Escalate-only (단방향 격상)**: 빠른 트리거는 레짐을 **더 방어적으로만** 바꾼다.
   절대 덜 방어적으로(예: BEAR→BULL) 되돌리지 않는다.
   `최종레짐 = max_defensiveness(느린레짐, 빠른트리거레짐)`.
   방어 등급: `BULL(0) < CAUTION(1) < BEAR(2) < CRISIS(3)`.
2. **Whipsaw 방지 — 인버스는 "지속" 조건에서만**:
   - **1일 단발 급락**은 **CAUTION(방어: 현금·채권)** 까지만. 인버스 매수 안 함.
   - **인버스(BEAR)는 3일 누적 지속 급락**에서만 허용 → 하루 급락 후 반등 시 인버스 손실 회피.
   - **1일 극단 급락은 CRISIS(현금)** → 가장 방어적이되 인버스 대신 현금(떨어지는 칼날 회피).
3. **Config 기반 임계값**: `strategy.yaml`의 `rapid_decline` 섹션에서 조정. 끄기 가능(`enabled`).

## 레벨 판정 (기본 임계값, 보수적)
| 트리거 레벨 | 조건 (지수 수익률) | 효과 |
|---|---|---|
| **CRISIS** | 1일 ≤ -8% **또는** 3일 ≤ -12% | 현금 70%+단기채 (인버스도 회피) |
| **BEAR** | 3일 ≤ -8% (지속 급락) | 인버스 1x 돌파 허용 + 채권 |
| **CAUTION** | 1일 ≤ -3% **또는** 3일 ≤ -5% | 롱 축소 + 현금·채권 (인버스 X) |
| **NONE** | 그 외 | 영향 없음(느린 레짐 그대로) |

> 단발 -5%는 CAUTION(방어)일 뿐 인버스를 사지 않는다. 인버스는 오직 3일 지속(-8%)에서만.
> 단발 -8%는 CRISIS(현금) — 인버스가 아니라 현금으로 가장 강하게 방어.

## 통합 위치
`detect_market_regime` 내부, HMM 보정 직후. 느린 레짐이 정해진 뒤
`rapid = detect_rapid_decline(index_history, cfg)` 결과와 escalate-only로 병합.
`detail`에 격상 사유를 남겨 대시보드/로그에서 추적 가능.

## 안전성
- escalate-only라 **최악의 경우도 "과하게 방어"** (현금·채권으로 기회비용) — 자본 손실 아님.
- 인버스 매수는 ① 3일 지속 급락 ② 기존 인버스 돌파 신호 ③ TA 확인 ④ 1x만(레버리지 금지)
  의 다중 게이트를 모두 통과해야 발생.
- CLAUDE.md #7(백테스트 맹신 금지): 라이브 반영 후 **모의·소액 관찰 필요**.
  초기 임계값은 보수적으로 두고, 실거래 관찰로 조정.
